This document summarizes the GO/NO-GO Status for the release of Rocky Linux 8.6 from the perspective of QA/Testing efforts. It is based largely on the Release Criteria (<https://wiki.rockylinux.org/team/testing/release_criteria/>), since it was originally imported from that document. If there are differences between the official Release Criteria document and this document, the official Release Criteria document will override, and this document shall be updated.

As a reminder, the objective of a release (major or minor) is to provide a solid Enterprise Linux release that meets the needs:

- Of end users
- Of enterprises, big or small

## SOP

In this document, each requirement is described, and the status is specified in the title.

Current choices are...

### TO_CONFIRM

- this means the item may be **[INCOMPLETE]**, **[PASS]**, **[FAIL_NON_BLOCKING[]** or **[FAIL_BLOCKING]** and must be verified

### PASS

- This means that the release criteria have been met and is not a blocker

### FAIL_NON_BLOCKING

- This means the release criterion has not been met, but is non-blocking

### FAIL_BLOCKING

- This means the release criteria have not been met and are blocking

In this document, the criteria status should include who completed the item and generally how it was completed.

Examples...

- **[PASS]** - (@tcooper, virt only, manual)
- **[PASS]** - (@lumarel, @raktajino, @tcooper, semi-automatic , openQA)
