## Versionshinweise für Rocky Linux 9.8

## Upgrade

Sie können von Rocky Linux 9.x auf Rocky Linux 9.8 aktualisieren, indem Sie `sudo dnf -y upgrade` ausführen.

!!! Note

    Rocky Linux bietet keinen Upgrade-Pfad von irgendeiner Version von Rocky Linux 8 an. Wir empfehlen Ihnen, eine Neuinstallation des Betriebssystems durchzuführen, um auf Rocky Linux 9.8 zu aktualisieren.

## Installation

Vor der Installation [stellen Sie sicher, dass Ihre CPU mit diesem Verfahren kompatibel ist!](https://docs.rockylinux.org/gemstones/test_cpu_compat/)

Um Rocky Linux 9.8 zu installieren, besuchen Sie die [Download-Seite](https://rockylinux.org/download/) und laden Sie die Version herunter, die Ihrer Architektur entspricht.

## Wichtigste Änderungen

Dieses Dokument hebt die wesentlichen Änderungen zwischen Rocky Linux (RL) 9.7 und RL 9.8 hervor. Falls dies Ihre erste Migration auf RL 9 ist, sollten Sie die Release Notes früherer Versionen durchgehen, um ein Gefühl für all die Änderungen zu bekommen, auf die Sie stoßen werden.

Eine Liste der wichtigsten Änderungen finden Sie in der [Upstream-Übersicht hier](https://docs.redhat.com/en/documentation/red_hat_enterprise_linux/9/html/9.8_release_notes/overview).

### Sicherheit

Nachfolgend sind die sicherheitsrelevanten Highlights der neuesten Version RL 9.8 aufgeführt:

- GnuTLS 3.8.10 bietet hybriden ML-KEM-Schlüsselaustausch sowie ML-DSA-Post-Quanten-Algorithmen (PQ)
- OpenSSH 9.9 mit umfangreichen Verbesserungen gegenüber OpenSSH 8.7
- Die `p11-kit`-Pakete wurden auf die Upstream-Version 0.26.1 aktualisiert, was Unterstützung für Definitionen der Post-Quanten-Kryptografie (PQC) in PKCS-#11-Headern bietet
- Das Paket `clevis-pin-trustee` stellt einen neuen Clevis-Pin-Trustee bereit, der die automatische Ver- und Entschlüsselung von LUKS-verschlüsselten Volumes mittels Remote-Attestierung über den Trustee Key Broker Service (KBS) ermöglicht.
- Die `fapolicyd`-Pakete wurden auf die Upstream-Version 1.4.3 aktualisiert und stellen nun Filterregeln bereit

### Kernel

- Erweitert die Kernel-Beobachtbarkeit für zusätzliche `perf`-Funktionen und neue Intel-Core-, Uncore-, C-State- und Package-Performance-Ereignisse
- Aktiviert die Unterstützung für `debuginfod` durch die Aktualisierung von `perf` auf neuere Upstream-Versionen und verbessert so die Abstimmung mit `perf` sowie den BPF-Tools
- Erweiterung der Leistungsindikatoren `uncore` und `core` für neuere Intel-Plattformen und Hinzufügen der Lastlatenzfilterung von AMD IBS, wodurch die CPU- und Speicheranalyse verbessert wird
- Verbessert die Hardware-Abdeckung durch das Hinzufügen oder Aktualisieren von Gerätetreibern und Geräte-IDs für Intel EDAC, Intel QAT sowie Intel/AMD-Beschleuniger- und Krypto-Geräte
- Aktualisiert das Kernel-Debugging und die Crash-Analyse durch die Neubasierung von `crash` sowie die Verbesserung der LUKS-fähigen kdump-Behandlung – sowohl im Kernel als auch in den kdump-Dienstprogrammen

## Bekannte Issues

Wie bei jeder Version gibt es einige bekannte Issues. Die vollständigen Details zu diesen Problemen finden Sie in der [Upstream-Liste](https://docs.redhat.com/en/documentation/red_hat_enterprise_linux/9/html/9.8_release_notes/known-issues).

!!! info "Veraltete Software"

    Einige Application Streams, die Teil der Repositories von Rocky Linux 9.8 sind, werden keine weiteren Updates erhalten. Dies ist ein regelmäßiges Vorkommnis, das bei fast jeder Veröffentlichung auftritt. Bitte stellen Sie sicher, dass Sie die neueste unterstützte Version der Software verwenden.
