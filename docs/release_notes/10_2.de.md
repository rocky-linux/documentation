## Rocky Linux 10.2

Rocky Linux 10.2 (RL 10.2) ist das neueste Minor-Release des Enterprise-Betriebssystems von Rocky Linux. In diesem Dokument sind die **bemerkenswertesten** Änderungen zwischen Version 10.1 und 10.2 aufgeführt. Wenn etwas nicht aufgeführt ist, bedeutet dies wahrscheinlich, dass es sich seit 10.1 kaum verändert hat. Falls Sie erstmals auf Rocky Linux 10 migrieren, sollten Sie sich möglicherweise auch die Release Notes für [10.0](10_0.md) und [10.1](10.1.md) ansehen. Aus diesen früheren Versionen sind einige kritische Anmerkungen verblieben.

### Upgrade auf Rocky Linux 10.2

Um von einer beliebigen Version von RL 10 auf 10.2 zu aktualisieren, führen Sie einfach das Kommando `dnf -y upgrade` aus.

Rocky Linux unterstützt keine Upgrades auf eine neue Hauptversion. Um von Version 8.x oder 9.x auf Rocky Linux 10.x zu wechseln, führen Sie eine Neuinstallation des Betriebssystems durch und stellen Sie die Daten aus einem Backup wieder her.

### Neue Funktionen und Änderungen

Die folgenden Abschnitte beschreiben ausgewählte Funktionen von RL 10.2.

Wie bei allen Rocky Linux-Veröffentlichungen umfassen diese Release-Notes zahlreiche Verbesserungen, Änderungen und neue Funktionen.

Eine vollständige Liste der wesentlichen Änderungen finden Sie in der Upstream-Übersicht [hier](https://docs.redhat.com/en/documentation/red_hat_enterprise_linux/10/html/10.2_release_notes/overview).

### Unterstützte Architekturen

!!! warning "Bitte überprüfen Sie die CPU-Kompatibilität"

    Auf der x86_64-Prozessorarchitektur unterstützt RL 10 keine Hardware der Mikroarchitektur-Ebene x86-64-v2 und früher mehr, sondern legt die Mikroarchitektur-Ebene x86-64-v3 als Basis für die Distribution fest. Um die Kompatibilität der Mikroarchitektur Ihrer CPU zu testen, [verwenden Sie dieses Verfahren](../gemstones/test_cpu_compat.md).

### Infrastruktur-Dienste

- PostgreSQL 18-Pakete sind jetzt verfügbar.
- MariaDB 11.8-Pakete sind jetzt verfügbar.
- Verfügbarkeit von PHP 8.4
- Die `chrony`-Pakete wurden auf Version 4.8 aktualisiert.
- Die `frr`-Pakete wurden auf Version 10.4.1 aktualisiert.

### Kernel

- Erweiterung der Kernel-Beobachtbarkeit um zusätzliche `perf`-Funktionen und neue Intel-Core-, Unicore-, C-State- und Package-Performance-Ereignisse.
- Eine engere Angleichung der `perf`- und BPF-Tools an das Upstream-Projekt, einschließlich der Aktualisierung von `perf` auf die Upstream-Version und der Aktivierung der `debuginfod`-Unterstützung.
- Erweiterung der Leistungsindikatoren `uncore` und `core` für neuere Intel-Plattformen und Hinzufügen der Lastlatenzfilterung von AMD IBS, wodurch die CPU- und Speicheranalyse verbessert wird.
- Fügt Treiber und Geräte-IDs für Intel EDAC, Intel QAT sowie Intel/AMD-Beschleuniger- und Kryptogeräte hinzu oder aktualisiert diese, wodurch die Hardware-Abdeckung verbessert wird.
- Erweitert die Schwellenwert-Überlaufaktionen von `rtla`, fügt Python-Bindings für `cpupower` hinzu und aktualisiert `rteval`, wodurch die Echtzeitanalyse und -optimierung verbessert werden.
- Aktualisiert das Kernel-Debugging und die Crash-Analyse durch die Neubasierung von `crash` sowie die Verbesserung der LUKS-fähigen kdump-Behandlung – sowohl im Kernel als auch in den kdump-Dienstprogrammen.

### Sicherheits-Features

- `keylime-agent` wurde auf Version 0.2.9 aktualisiert
- Die `fapolicyd`-Pakete wurden auf Version 1.4.3 aktualisiert und verfügen nun über eine Regelfilterfunktion
- Das Paket `clevis-pin-trustee` stellt einen neuen Clevis-PIN-Trustee bereit, der die automatisierte Ver- und Entschlüsselung von LUKS-verschlüsselten Volumes durch Remote-Attestierung über den Trustee Key Broker Service (KBS) ermöglicht.
- `capnproto` ist ein neues Paket, das ein leistungsstarkes System für den Datenaustausch und Remote Procedure Calls (RPC) bereitstellt und dabei Zero-Copy-Serialisierung nutzt, um den Overhead herkömmlicher Kodierungs- und Dekodierungsprozesse zu eliminieren.
- Das Paket `openssh` führt den Modulgitter-basierten Schlüsselkapselungsmechanismus (ML-KEM) für den Post-Quanten-Schlüsselaustausch (PQ) in Kombination mit elliptischen Kurven ein, die vom National Institute of Standards and Technology (NIST) im FIPS-Modus standardisiert wurden.
- `libssh` führt Unterstützung für hybride Post-Quanten-Schlüsselaustauschverfahren (PQ/T) ein, die auf dem quantenresistenten ML-KEM-Standard sowie traditionellen ECDH-Schlüsselaustauschverfahren (Elliptic-curve Diffie-Hellman) basieren.
- Die `p11-kit`-Pakete wurden auf die Upstream-Version 0.26.1 aktualisiert und bieten nun Unterstützung für Definitionen der Post-Quanten-Kryptografie (PQC) in PKCS-#11-Headern.
- Die Bibliothek `podman-sequoia` unterstützt zusammengesetzte Post-Quantum-Signaturen.

## Bekannte Issues

Da 10.2 nun zwei PHP-Versionen (8.3 und 8.4) enthält, kann das bloße Hinzufügen einer Abhängigkeit Probleme verursachen. Stellen Sie bei der Installation von PHP-Abhängigkeiten sicher, dass die Abhängigkeit mit Ihrer installierten Version übereinstimmt. Wenn Sie beispielsweise `php-json` installieren, führen Sie `dnf provides php-json` aus und installieren Sie anschließend – basierend auf der Ausgabe – das Paket, das für Ihre installierte PHP-Version passend ist.

Es gibt [bekannte Issues, die im Upstream hier gemeldet wurden.](https://docs.redhat.com/en/documentation/red_hat_enterprise_linux/10/html/10.2_release_notes/known-issues)

!!! info "Veraltete Software"

    Einige Application Streams, die Teil der Repositories von Rocky Linux 10.2 sind, werden keine weiteren Updates erhalten. Dies ist ein regelmäßiges Vorkommnis, das bei fast jeder Veröffentlichung auftritt. Bitte stellen Sie sicher, dass Sie die neueste unterstützte Version der Software verwenden.
