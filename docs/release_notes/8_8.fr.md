# Notes de version de Rocky Linux 8.8

## Mise à niveau

Vous pouvez mettre à niveau depuis Rocky Linux 8.x vers Rocky Linux 8.8 en exécutant la commande `sudo dnf -y`.

## Images

En plus des images d'installation classiques, plusieurs images sont disponibles pour de nombreuses plateformes cloud et de conteneurs.

Des images pour Oracle Cloud Platform (OCP), GenericCloud, Amazon AWS (EC2), Container RootFS/OCI, Google Cloud Platform, Microsoft Azure, et d'autres images maintenues par des CSP sont disponibles. Les images GenericCloud, EC2 et Microsoft Azure disposent d'une variante utilisant une partition LVM pour le système de fichiers racine, offrant aux administrateurs système une flexibilité supplémentaire et plus d'options pour configurer leurs systèmes.

Les images vagrant et conteneurs sont disponibles pour x86_64 et aarch64. Depuis la dernière version, nous avons ajouté la variante `rockylinux/rockylinux:8-ubi-init` pour l'utilisation de systemd dans un conteneur. Nous avons également ajouté la variante `rockylinux/rockylinux:8-ubi-micro`, qui est l'image la plus minimaliste pouvant être utilisée dans des applications autonomes sans gestionnaire de paquets. Nous avons aussi ajouté des images [Toolbox](https://containertoolbx.org/) pour davantage d'architectures.

Vous trouverez des informations sur les artefacts produits par le groupe Cloud Special Interest Group, ainsi que des informations sur les façons de s'impliquer, sur la page [SIG/Cloud Wiki](https://sig-cloud.rocky.page/).

Les images live suivantes sont à nouveau disponibles pour x86_64 : Workstation, Workstation Lite, KDE, XFCE, MATE.

## Installation

Pour installer Rocky Linux 8.8, rendez-vous sur la [page de téléchargement](https://rockylinux.org/download/) et téléchargez la version correspondant à votre architecture.

## Points forts de la version par l'équipe Rocky

### Changements majeurs

- Les images Microsoft Azure sont maintenant publiées dans la Galerie d'images partagées ainsi que sur le Marketplace. La Galerie d'Images Partagées offre un moyen direct d'utiliser les images Rocky sur Microsoft Azure sans avoir à passer par les étapes d'abonnement via le Marketplace. De plus, cette Galerie d'Images Partagées nous permettra de publier des images plus fréquemment mises à jour sur le Marketplace.
- Les variantes d'images cloud LVM pour tous les types suppriment désormais `/etc/lvm/devices/system.devices` afin de résoudre les problèmes avec les PV/VG/LV lors de l'installation des images, dus à un chemin de périphérique codé en dur.
- L'**image de base universelle pour conteneurs** `rockylinux/rockylinux:8-ubi` a été modifiée pour ressembler davantage aux images UBI RHEL. Les changements suivants ont été effectués :

  - Paquets remplacés : `libcurl` -> `curl-minimal`, `libcurl-minimal`
  - Paquets ajoutés : `gdb-gdbserver`, `gzip`
  - Paquets supprimés : `binutils`, `brotli`, `dmidecode`
  - nettoyage de la configuration réseau ajouté

## Changements majeurs

Pour une liste complète des changements majeurs, veuillez consulter la liste [en amont ici](https://access.redhat.com/documentation/en-us/red_hat_enterprise_linux/8/html/8.8_release_notes/overview#overview-major-changes).

Quelques points forts et nouvelles fonctionnalités de cette version sont détaillés ci-après.

### Programme d'installation et création d'images

- Image builder propose désormais une nouvelle méthode améliorée pour créer des plans et des images dans la console web d'image builder.

### Sécurité

Les points forts liés à la sécurité dans la dernière version de Rocky Linux 8.8 sont listés ci-dessous. Pour une liste complète des changements liés à la sécurité, consultez le lien [en amont ici](https://access.redhat.com/documentation/en-us/red_hat_enterprise_linux/8/html/8.8_release_notes/new-features#new-features-security).

- Les paramètres du mode FIPS dans le noyau ont été ajustés pour se conformer à la norme Federal Information Processing Standard (FIPS) 140-3. Ce changement introduit des paramètres plus stricts pour de nombreux algorithmes cryptographiques, fonctions et suites de chiffrement.
- Les paquets `libreswan` ont été rebasés sur la version 4.9.
- Vous pouvez désormais filtrer la base de données RPM avec les paquets `fapolicyd`.
- Les paquets `openscap` ont été rebasés vers la version 1.3.7.
- Le pilote OpenSSL peut désormais utiliser des chaînes de certificats dans Rsyslog.
- Le service `systemd-socket-proxyd` s'exécute désormais dans son propre domaine SELinux suite à une mise à jour de la politique SELinux.

### Langages de programmation dynamiques, serveurs web et de bases de données

Pour une liste détaillée des changements dans cette catégorie, consultez le lien [en amont ici](https://access.redhat.com/documentation/en-us/red_hat_enterprise_linux/8/html/8.8_release_notes/new-features#new-features-dynamic-programming-languages-web-and-database-servers).

Des versions plus récentes des Application Streams suivants sont désormais disponibles :

- nginx 1.22
- PostgreSQL 15
- Jakarta XML Binding (`jaxb`) 4
- Swig 4.1

Les applications suivantes ont été ajoutées :

- Python 3.11
- Tomcat 9

Les composants suivants ont été ajoutés ou mis à jour :

- Git vers la version 2.39.1 :

  - La signature de commits avec des clés SSH est désormais prise en charge

- Git LFS vers la version 3.2.0

### Compilateurs et outils de développement

Pour les détails sur les changements dans cette catégorie, consultez le lien [en amont ici](https://access.redhat.com/documentation/en-us/red_hat_enterprise_linux/8/html/8.8_release_notes/new-features#new-features-compilers-and-development-tools).

- LLVM Toolset 15.0.7
- Rust Toolset 1.66
- Go Toolset 1.19.4

#### Outils de performance et débogueurs mis à jour

Les outils de performance et les débogueurs suivants ont été mis à jour :

- Valgrind 3.19
- SystemTap 4.8
- elfutils 0.188

#### Outils de surveillance des performances mis à jour

Les outils de surveillance des performances suivants ont été mis à jour :

- PCP 5.3.7
- Grafana 7.5.15

### Conteneurs

Pour les détails sur les changements dans cette catégorie, consultez le lien [amont ici](https://access.redhat.com/documentation/en-us/red_hat_enterprise_linux/8/html/8.8_release_notes/new-features#new-features-containers). Parmi les changements notables :

- Les rôles système Linux pour podman sont désormais disponibles
- Les clients pour les signatures sigstore avec Fulcio et Rekor sont désormais disponibles
- Skopeo prend désormais en charge la génération de paires de clés sigstore
- Podman prend désormais en charge les événements pour l'audit
- Les paquets Container Tools ont été mis à jour
- La pile réseau Aardvark et Netavark prend désormais en charge la sélection de serveurs DNS personnalisés
- Podman Quadlet est désormais disponible en aperçu technologique

### Infrastructures graphiques

- La prise en charge des GPU Intel ARC est désormais disponible en aperçu technologique
